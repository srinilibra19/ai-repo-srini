Decisions made:
  - Solace host uses `tcps://localhost:55443` (mTLS, per backlog AC) — not
    plaintext 55555. Both ports are exposed in docker-compose.yml.
  - `authentication-scheme: CLIENT_CERTIFICATE` added explicitly; Starter does
    not auto-switch from basic auth to mTLS just because a keystore is present.
  - `client-name: hermes-flightschedules-local-${HOSTNAME:dev}` — includes
    destination segment to prevent session takeover when 2 devs share a VPN.
  - DataSource URL: `?sslmode=prefer` added to avoid plaintext default; local
    Docker container accepts prefer (no SSL), real RDS upgrades automatically.
  - All credential values wrapped in `${ENV_VAR:default}` — no naked literals.
  - Credential provider design documented in YAML comments for AwsConfig.java:
    non-blank hermes.aws.credentials.* → StaticCredentialsProvider (local only);
    absent → DefaultCredentialsProvider (IRSA on ROSA).
  - Spring binding/condition log suppressed to prevent password values appearing
    in DEBUG startup output.
